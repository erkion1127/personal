# Claude 문서 관리 규칙

이 문서는 Claude가 이 프로젝트의 문서를 수정하거나 관리할 때 따라야 할 규칙을 정의합니다.

## 기본 원칙

1. **개인정보 보호 최우선**
   - 민감한 개인정보(주민등록번호, 계좌번호, 비밀번호 등)는 절대 노출하지 않음
   - 문서 내용을 요약하거나 설명할 때 개인정보는 생략하거나 마스킹

2. **문서 분류 체계 유지**
   - 기존 폴더 구조를 준수
   - 새 문서 추가 시 적절한 카테고리에 배치
   - 불명확한 경우 사용자에게 확인 요청

3. **파일명 규칙**
   - 날짜 포함 시: `YYYY-MM-DD_설명.확장자`
   - 설명적이고 명확한 이름 사용
   - 한글 사용 가능, 공백 대신 하이픈(-) 또는 언더스코어(_) 사용

## 폴더별 관리 규칙

### personal/ (개인 문서)

#### family/
- 가족 관련 모든 문서
- 민감 정보 포함 가능성 높음 → 특별히 주의

#### certificates/
- 졸업증명서, 자격증, 면허증 등
- 원본 유지, 수정 금지
- 복사본 생성 시 `-copy` 접미사 추가

#### financial/
- **stock-tax/**: 주식 관련 세금 문서
- **stock/**: 주식 보유 및 거래 관련
- **stock-documents/**: 기타 주식 문서
- **bank-interest/**: 은행 이자 내역
- 금융 정보는 절대 외부 공유 금지
- 요약 시 금액은 대략적인 범위로만 표현

#### real-estate/
- **mokpo/**: 목포 부동산 관련 (영업신고, 급여, 세금 등)
- **gwanggyo/**: 광교 부동산 관련
- 사업자등록증, 계약서 등 법적 문서 포함
- 원본 보존 필수

#### health/
- 건강검진 결과, 의료비 영수증
- 민감한 의료정보 포함 → 주의

#### vehicle/
- 차량 관련 문서

#### career/
- 이력서, 면접 준비 자료
- 회사별로 하위 폴더 구성

### work/ (업무 문서)

#### notes/
- 일반 업무 노트, 메모
- 회의록은 meetings/ 폴더로

#### meetings/
- 회의록 작성 시 `templates/meeting-notes.md` 사용
- 날짜별로 파일명 시작: `YYYY-MM-DD-회의제목.md`

#### reports/
- 보고서, 분석 자료
- 완료된 프로젝트는 archive/로 이동

#### archive/
- 완료되었거나 더 이상 사용하지 않는 문서

#### settings/
- 개발 도구 설정 파일
- 수정 시 백업 생성 권장

### policies/ (정책 문서)

- MCP 서버에서 참조하는 정책 문서
- 수정 시 버전 관리 필수
- 중요한 변경사항은 changelog 작성

### templates/ (템플릿)

- 문서 작성 템플릿
- 수정 시 기존 템플릿은 유지하고 새 버전 생성

## 문서 수정 시 체크리스트

- [ ] 개인정보가 노출되지 않는지 확인
- [ ] 올바른 폴더에 저장되었는지 확인
- [ ] 파일명이 규칙에 맞는지 확인
- [ ] 중요 문서는 원본 백업 생성
- [ ] 변경사항을 명확한 커밋 메시지로 기록

## Git 커밋 규칙

### 커밋 메시지 형식
```
[카테고리] 간단한 설명

- 상세 내용 1
- 상세 내용 2

관련 이슈: #이슈번호 (있는 경우)
```

### 카테고리
- `[personal]`: 개인 문서 추가/수정
- `[work]`: 업무 문서 추가/수정
- `[policy]`: 정책 문서 추가/수정
- `[template]`: 템플릿 추가/수정
- `[structure]`: 폴더 구조 변경
- `[docs]`: 문서 설명 추가/수정

### 예시
```
[personal] 2024년 건강검진 결과 추가

- personal/health/건강검진/20241224_건강검진.pdf 추가
- 정기 건강검진 결과 문서화
```

## 보안 및 프라이버시

### 절대 커밋하지 말아야 할 정보
- 주민등록번호, 여권번호
- 신용카드 번호, 계좌번호
- 비밀번호, API 키, 토큰
- 서명 이미지, 인감

### .gitignore 규칙
- `*.private.md`: 개인적인 메모
- `*.private.*`: 개인 파일
- `.DS_Store`: macOS 시스템 파일

## Claude 사용 시나리오

### 문서 검색
```
"2024년 건강검진 결과 찾아줘"
→ personal/health/건강검진/ 폴더 검색
```

### 문서 요약
```
"목포 영업신고 관련 문서 요약해줘"
→ personal/real-estate/mokpo/영업신고/ 문서 요약
→ 개인정보는 마스킹하여 제공
```

### 회의록 작성
```
"오늘 회의록 작성해줘"
→ templates/meeting-notes.md 복사
→ work/meetings/YYYY-MM-DD-제목.md 생성
```

### 문서 정리
```
"개인 문서 중복 파일 찾아줘"
→ 파일명, 크기, 날짜 비교
→ 중복 가능성 있는 파일 리스트 제공
```

## 참고사항

- 문서 수정 전 항상 사용자 확인 요청
- 불명확한 경우 안전한 선택 (보수적 접근)
- 정기적으로 문서 구조 검토 및 최적화 제안
