# 문서 관리 정책

## 목적

이 정책은 개인 및 업무 문서의 체계적인 관리를 위한 지침을 제공합니다.

## 문서 분류 체계

### 1. 개인 문서 (personal/)

개인적인 문서는 카테고리별로 분류하여 관리합니다.

#### 분류 기준
- **family**: 가족 관련 모든 문서
- **certificates**: 학위, 자격증, 면허증 등 공식 증명서
- **financial**: 금융 관련 문서 (세금, 주식, 은행)
- **real-estate**: 부동산 관련 문서 (위치별로 하위 분류)
- **health**: 건강검진, 의료비 관련
- **vehicle**: 차량 관련
- **career**: 이력서, 면접, 경력 관련

#### 보존 기간
- **영구 보존**: certificates/, real-estate/ 내 계약서/등기 관련
- **10년**: financial/ 내 세금 관련
- **5년**: health/ 내 의료비 영수증
- **3년**: vehicle/ 관련 문서
- **필요시까지**: career/, family/

### 2. 업무 문서 (work/)

업무 관련 문서는 진행 상태와 중요도에 따라 관리합니다.

#### 분류 기준
- **notes**: 일상적인 업무 메모
- **meetings**: 회의록
- **reports**: 보고서, 분석 자료
- **archive**: 완료된 프로젝트 문서
- **settings**: 개발 환경 설정

#### 정리 주기
- 분기별로 완료된 프로젝트를 archive/로 이동
- 6개월 이상 사용하지 않은 notes는 archive/ 검토

### 3. 정책 문서 (policies/)

MCP 서버 및 AI 어시스턴트가 참조하는 정책 문서입니다.

#### 관리 규칙
- 모든 변경사항은 버전 관리
- 중요 변경 시 changelog 작성
- 정책 문서는 마크다운(.md) 형식 사용

### 4. 템플릿 (templates/)

재사용 가능한 문서 템플릿을 관리합니다.

#### 템플릿 관리
- 템플릿 수정 시 기존 버전 유지
- 새 버전은 `-v2`, `-v3` 등으로 구분
- 사용 빈도가 높은 템플릿 우선 관리

## 파일명 규칙

### 기본 형식
```
YYYY-MM-DD_카테고리_설명.확장자
```

### 예시
- `2024-12-24_회의록_프로젝트킥오프.md`
- `2024-12-24_보고서_분기실적.pdf`
- `2024-12-24_건강검진.pdf`

### 규칙
1. 날짜 형식: YYYY-MM-DD (필요한 경우만)
2. 한글 사용 가능
3. 공백 대신 언더스코어(_) 또는 하이픈(-) 사용
4. 소문자 사용 권장 (한글 제외)
5. 특수문자 사용 금지 (!, @, #, $ 등)

## 보안 및 프라이버시

### 민감 정보 처리

#### 민감 정보 분류
- **Level 1 (최상위)**: 주민등록번호, 여권번호, 계좌번호, 비밀번호
- **Level 2 (상위)**: 급여명세서, 세금 신고서, 의료 기록
- **Level 3 (중위)**: 계약서, 자격증, 건강검진
- **Level 4 (하위)**: 일반 업무 문서, 회의록

#### 보안 조치
- Level 1-2 문서는 암호화된 저장소 고려
- 파일명에 민감 정보 포함 금지
- Git 커밋 전 민감 정보 확인

### .gitignore 정책

다음 파일들은 Git에 커밋하지 않습니다:
```
# 개인 파일
*.private.md
*.private.*

# 시스템 파일
.DS_Store
Thumbs.db

# 임시 파일
*.tmp
*.bak
*~

# IDE
.idea/
.vscode/
```

## 백업 정책

### 백업 주기
- **일일**: 진행 중인 중요 문서 (work/reports/, work/meetings/)
- **주간**: 개인 문서 전체 (personal/)
- **월간**: 전체 프로젝트 백업

### 백업 방법
1. GitHub 원격 저장소 (이미 설정됨)
2. 로컬 외장 디스크
3. 클라우드 스토리지 (선택사항)

### 복구 절차
1. Git을 통한 버전 복구
2. 로컬 백업에서 복구
3. 클라우드에서 복구

## 문서 생명주기

### 1. 생성
- 적절한 카테고리 선택
- 파일명 규칙 준수
- 메타데이터 추가 (날짜, 작성자 등)

### 2. 사용
- 정기적으로 검토 및 업데이트
- 변경사항 커밋

### 3. 보관
- 완료된 문서는 archive/로 이동
- 보존 기간에 따라 관리

### 4. 폐기
- 보존 기간 만료 시 검토
- 중요 문서는 영구 보존
- 폐기 시 로그 기록

## AI 어시스턴트 사용 지침

### Claude/ChatGPT 사용 시

#### 허용되는 작업
- 문서 검색 및 요약
- 회의록 작성 지원
- 문서 템플릿 생성
- 파일명 정리 제안
- 문서 구조 최적화 제안

#### 제한되는 작업
- 민감 정보 포함 문서를 외부 API로 전송
- 사용자 확인 없이 문서 삭제
- 정책 문서 임의 수정

#### 사용 예시
```
사용자: "2024년 세금 관련 문서 찾아줘"
AI: personal/financial/stock-tax/ 폴더 검색
    → 파일 리스트 제공 (금액 등 민감정보 제외)

사용자: "회의록 작성해줘"
AI: templates/meeting-notes.md 복사
    → work/meetings/YYYY-MM-DD-제목.md 생성
    → 사용자가 내용 작성
```

## MCP 서버 연동

### 설정 방법

MCP 서버 설정 파일에 다음 추가:

```json
{
  "mcpServers": {
    "jslee27-docs": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/ijeongseob/Documents/doc/jslee27"
      ]
    }
  }
}
```

### 참조 가능한 문서
- `policies/`: AI가 참조할 정책 문서
- `CLAUDE_RULES.md`: Claude 사용 규칙
- `README.md`: 프로젝트 개요

### 제한 사항
- `personal/` 폴더 내 민감 정보는 AI가 직접 접근하지 않도록 설정
- 필요 시 사용자가 명시적으로 요청한 경우만 접근
