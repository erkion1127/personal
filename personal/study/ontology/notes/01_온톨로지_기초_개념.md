# 온톨로지 기초 개념

## 온톨로지란?

**온톨로지(Ontology)**는 특정 도메인의 개념과 개념 간의 관계를 형식적으로 표현한 것입니다.

### 철학적 정의
- 존재론: "무엇이 존재하는가?"를 다루는 철학 분야
- 실재하는 것들의 분류 체계

### 컴퓨터 과학적 정의
> "A formal, explicit specification of a shared conceptualization"
> - Thomas Gruber (1993)

**풀이**:
- **Formal**: 컴퓨터가 처리 가능한 형식
- **Explicit**: 명시적으로 정의된
- **Shared**: 공유 가능한
- **Conceptualization**: 개념화

## 왜 온톨로지가 필요한가?

### 1. 지식 공유
- 여러 시스템/사람 간에 공통된 이해
- 표준화된 어휘 사용

### 2. 지식 재사용
- 한 번 정의한 온톨로지를 여러 곳에서 활용
- 도메인 지식의 축적

### 3. 자동 추론
- 명시적 지식에서 암묵적 지식 유도
- 논리 규칙 기반 추론

### 4. 데이터 통합
- 이질적인 데이터 소스 통합
- 의미 기반 검색

## 온톨로지 vs 다른 개념들

### 온톨로지 vs 데이터베이스 스키마

| 특징 | 온톨로지 | DB 스키마 |
|------|----------|-----------|
| 목적 | 지식 표현 | 데이터 저장 |
| 추론 | 가능 | 제한적 |
| 개방성 | Open World Assumption | Closed World |
| 유연성 | 높음 | 낮음 |
| 의미론 | 풍부 | 제한적 |

**Open World vs Closed World**:
- **Open World**: 명시되지 않은 것은 "모른다" (Unknown)
- **Closed World**: 명시되지 않은 것은 "거짓" (False)

### 온톨로지 vs 택소노미(Taxonomy)

- **택소노미**: 단순 계층 구조 (is-a 관계만)
  ```
  동물
  ├── 포유류
  │   ├── 개
  │   └── 고양이
  └── 조류
  ```

- **온톨로지**: 다양한 관계 표현 가능
  ```
  개 is-a 포유류
  개 has-part 꼬리
  개 eats 사료
  개 lives-with 사람
  ```

### 온톨로지 vs 지식 그래프

- **온톨로지**: 스키마, 개념 모델, 규칙
- **지식 그래프**: 온톨로지 + 인스턴스 데이터
  ```
  온톨로지: Person, Company, works-at (스키마)
  지식 그래프: Alice works-at Google (데이터)
  ```

## 온톨로지의 핵심 구성 요소

### 1. 클래스 (Class)
- 개념의 집합
- 예: Person, Animal, Vehicle

### 2. 인스턴스 (Instance)
- 클래스의 구체적 개체
- 예: "Alice", "Fluffy", "Tesla Model 3"

### 3. 속성 (Property)
- **객체 속성 (Object Property)**: 개체 간 관계
  - 예: hasParent, worksAt
- **데이터 속성 (Data Property)**: 개체와 값
  - 예: hasAge, hasName

### 4. 관계 (Relation)
- 클래스/인스턴스 간의 연결
- 예: subClassOf, instanceOf

### 5. 공리 (Axiom)
- 참으로 간주되는 명제
- 제약 조건, 규칙
- 예: "모든 사람은 정확히 두 명의 생물학적 부모를 가진다"

## 간단한 예제: 가족 온톨로지

### 클래스
```
Person
├── Male
└── Female
```

### 속성
```
hasParent (Person → Person)
hasChild (Person → Person)
hasSpouse (Person → Person)
hasName (Person → String)
hasAge (Person → Integer)
```

### 인스턴스
```
John: Male
  hasName: "John Smith"
  hasAge: 45

Mary: Female
  hasName: "Mary Smith"
  hasAge: 43

Bob: Male
  hasName: "Bob Smith"
  hasAge: 20
  hasParent: John
  hasParent: Mary
```

### 추론 규칙
```
규칙 1: hasParent의 역관계는 hasChild
  X hasParent Y  →  Y hasChild X

규칙 2: 조부모 관계
  X hasParent Y, Y hasParent Z  →  X hasGrandparent Z

규칙 3: 형제 관계
  X hasParent P, Y hasParent P, X ≠ Y  →  X hasSibling Y
```

### 추론 결과
```
명시적 지식: Bob hasParent John

추론된 지식:
- John hasChild Bob (규칙 1)
- Bob hasGrandparent [John의 부모] (규칙 2)
- Bob hasSibling [John의 다른 자녀들] (규칙 3)
```

## 온톨로지 사용 사례

### 1. 의료 분야
- SNOMED CT: 의료 용어 온톨로지
- 질병, 증상, 치료법 간 관계 표현

### 2. 전자상거래
- 상품 분류 및 추천
- 속성 기반 검색

### 3. 시맨틱 웹
- Schema.org: 웹 페이지 의미 표현
- 검색 엔진 최적화

### 4. 기업 지식 관리
- 조직 구조, 프로세스 모델링
- 문서 분류 및 검색

### 5. AI 에이전트
- 에이전트의 세계 모델
- 추론 기반 의사결정

## 핵심 요점 정리

1. 온톨로지는 **도메인 지식의 형식적 표현**
2. **자동 추론**을 통해 새로운 지식 발견
3. **표준화**를 통한 지식 공유 및 재사용
4. DB 스키마보다 **의미론적으로 풍부**
5. 지식 그래프의 **스키마 역할**
